## 🔄 Control de Cambios con Git

### Ver lo que hiciste
```powershell
git log --oneline
# Resultado:
# 2c40fb4 (HEAD -> main) docs: Add complete documentation and setup guides
# 2969900 (tag: v1.0.0) feat: Initial commit - EarnApp Reviewer Python v1.0.0
```

### Para cambios futuros (cuando hagas ediciones)
```powershell
git add .
git commit -m "feat: Descripción de lo que cambiaste"
git push origin main
```

### Para nuevas versiones
```powershell
echo "1.0.1" > VERSION
git add VERSION
git commit -m "chore: Bump to 1.0.1"
git tag -a v1.0.1 -m "Version 1.0.1"
git push origin main
git push origin v1.0.1
# GitHub Actions crea automáticamente la release!
```

## ❓ PREGUNTAS RÁPIDAS

**P: ¿Necesito instalar Python aparte?**  
R: Sí. Descárgalo desde https://www.python.org (versión 3.9+)

**P: ¿Por qué Python en lugar de PHP?**  
R: 
- GUI profesional (no web)
- No se congela durante operaciones
- Mejor para desktop
- Fácil de compilar a EXE

**P: ¿Cómo le doy a otros?**  
R: Sube a GitHub y comparte el link. Ellos descargan e instalan con `setup-env.bat`

**P: ¿Puedo cambiar el código?**  
R: Claro, está bien estructurado. Edita, prueba, commit, push.

**P: ¿Cómo hago EXE ejecutable?**  
R: 
```powershell
pip install pyinstaller
pyinstaller --onefile --windowed --icon=icon.ico main.py
# Output: dist\main.exe
```
